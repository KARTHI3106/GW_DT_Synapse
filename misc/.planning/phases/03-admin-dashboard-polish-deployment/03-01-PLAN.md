<action>
Update `backend/routers/admin.py` `get_claims_forecast()`:

Replace the current historical-data-only approach with a hybrid forecast:

1. Fetch 7-day weather forecast from OpenWeatherMap via `integrations.weather.fetch_7day_forecast(city)` for each city
2. For each day, calculate trigger probabilities:
   - rain_trigger_prob = min(day.rain / 30.0, 1.0) if day has rain field
   - heat_trigger_prob = 1.0 if day.feels_like.day > 43 else (day.feels_like.day - 35) / 8 clamped to 0-1
   - aqi_trigger_prob = estimated from historical AQI patterns (0.3 for Delhi, 0.1 for Mumbai, 0.05 for Bengaluru as defaults)
3. Estimate claims per city per day: active_workers_in_city * highest_trigger_prob * 0.6
4. Estimate payout per day: estimated_claims * weighted_avg_payout (Rs.320)

Return array of forecast objects grouped by date:
```python
{
    "forecast": [
        {
            "date": "2026-04-04",
            "rain_trigger_prob": 0.45,
            "heat_trigger_prob": 0.0,
            "aqi_trigger_prob": 0.15,
            "estimated_claims": 4,
            "estimated_payout": 1280,
            "cities": {"Mumbai": {...}, "Delhi": {...}, "Bengaluru": {...}}
        }
    ]
}
```

Add try/except around weather API calls with fallback to historical averages if API fails.
</action>
