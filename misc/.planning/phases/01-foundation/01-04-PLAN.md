**`/backend/services/policy_service.py`** — Business logic:
```python
import hashlib
import random
import string
from datetime import date, timedelta

CITY_CODES = {"Mumbai": "MUM", "Delhi": "DEL", "Bengaluru": "BLR"}

def generate_policy_number(city: str) -> str:
    """Generate policy number: GS-2026-MUM-XXXXXX"""
    year = date.today().year
    city_code = CITY_CODES.get(city, "IND")
    suffix = "".join(random.choices(string.digits, k=6))
    return f"GS-{year}-{city_code}-{suffix}"

async def create_policy_for_worker(worker_id: str, worker_city: str, weekly_premium: int, coverage_amount: int, supabase) -> dict:
    """
    Auto-create policy on worker registration.
    Simulates Guidewire PolicyCenter policy lifecycle.
    [MOCK/SIMULATED] In production: POST to Guidewire PC REST API.
    """
    today = date.today()
    policy_data = {
        "policy_number": generate_policy_number(worker_city),
        "worker_id": worker_id,
        "start_date": today.isoformat(),
        "end_date": (today + timedelta(days=7)).isoformat(),
        "status": "active",
        "weekly_premium": weekly_premium,
        "coverage_amount": coverage_amount
    }
    result = supabase.table("policies").insert(policy_data).execute()
    return result.data[0]
```

**`/backend/routers/policies.py`** — Full implementation:

```python
- [MOCK/SIMULATED] Simulates Guidewire BillingCenter

- Sets status to 'paused'

POST /api/v1/policies/{policy_id}/resume
- Sets status back to 'active'
```

Update the registration endpoint in `workers.py` to call `create_policy_for_worker` after inserting the worker, and include `policy_number` and `weekly_premium` in the response.
