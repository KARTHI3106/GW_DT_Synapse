**`/backend/routers/workers.py`** — Full implementation:
```python
POST /api/v1/workers/register
- Validate phone format (10 digits)
- Hash phone with SHA256 before storage
- Check duplicate worker_id
- Insert into workers table via Supabase
- Auto-call policy creation service (from policies router)
- Return: {id, worker_id, policy_number, weekly_premium, message}

GET /api/v1/workers/{worker_id}
- Fetch worker by worker_id, join with active policy
- Return worker profile + policy summary
```

Connect to Supabase client from db/supabase.py. All exceptions caught and returned as structured error JSON: `{"status": "error", "error": {"code": "...", "message": "..."}}`

### Task 4: Wire premium preview in registration form

When user is on Step 3 (earnings input) — call `/api/v1/premiums/calculate` with current form data and display the result.

Since the premium API won't exist yet at this plan's wave, use a client-side estimate function while the real API wires up in the plan 01-03:

**`/frontend/lib/premium-estimate.ts`**:
```typescript
// Client-side estimate used as preview before XGBoost API is available
export function estimatePremium(city: string, zone: string, rating: number, hours: number, earnings: number): number {
  const BASE = 159
  const cityMultiplier = { Mumbai: 1.15, Delhi: 1.1, Bengaluru: 0.95 }[city] ?? 1.0
  const ratingBonus = rating >= 4.5 ? 0.85 : rating >= 4.0 ? 0.95 : 1.0
  const earningsCap = Math.min(earnings * 0.03, BASE * 2) // 3% affordability ceiling
  return Math.round(Math.min(BASE * cityMultiplier * ratingBonus, earningsCap))
}
```

Show estimate live. Once Plan 01-03 (real XGBoost API) is done, update to use the real endpoint.
