### Task 1: Create mock workers seed script

- backend/db/zones_seed.sql (available zones)
- backend/routers/workers.py (registration flow for field formats)
</read_first>

<action>
Create `backend/seeds/mock_workers.py`:

Python script that inserts 15 realistic worker profiles (5 per city) using the Supabase client. Each worker should have varied:

- Platforms: mix of Swiggy and Zomato
- Ratings: 3.2 to 4.9

### Task 2: Create mock claims seed for fraud model training

<read_first>
- backend/db/schema.sql (claims table schema)
- backend/ml/fraud_detector.py (features expected by IsolationForest)
</read_first>

<action>
Create `backend/seeds/mock_claims.py`:

Generate 60 synthetic claims spread across the 15 workers:
- 50 legitimate claims: fraud_score 0.05-0.25, status "paid", varied trigger types
- 10 suspicious claims: fraud_score 0.5-0.9, status "pending" or "rejected"
  - Some with zone mismatches (GPS fraud signal)
  - Some with timing anomalies (>3 hours after trigger)
  - Some duplicates (same trigger on same day)

Each claim should have realistic:
- claim_number: CL-2026-{type[:3].upper()}-{hash}
- trigger_timestamps spanning the last 30 days
- payout_amounts matching trigger type payouts
- transaction_ids: SIM-{hex} for paid claims

Insert fraud_flags for the suspicious claims.

This data enables the Isolation Forest to fit properly (needs 10+ samples).
</action>

<acceptance_criteria>
- backend/seeds/mock_claims.py exists
- backend/seeds/mock_claims.py creates at least 50 claims
- backend/seeds/mock_claims.py creates fraud_flags for suspicious claims
- Mix of legitimate (status=paid) and suspicious (status=pending/rejected) claims
</acceptance_criteria>
