Create `backend/services/payout_service.py`:

```python
import uuid
from datetime import datetime, timezone
from db.supabase import get_supabase

async def process_payout(claim_id: str, amount: int, worker_phone_hash: str) -> dict:
    """
    Simulate Razorpay UPI payout.
    [MOCK/SIMULATED] In production, uses RazorpayX Payout API:
    POST https://api.razorpay.com/v1/payouts
    """
    transaction_id = f"SIM-{uuid.uuid4().hex[:12].upper()}"
    phone_suffix = worker_phone_hash[-4:]
    
    payout_result = {
        "transaction_id": transaction_id,
        "amount": amount,
        "currency": "INR",
        "mode": "UPI",
        "upi_id": f"{phone_suffix}@upi",
        "status": "processed",
        "processed_at": datetime.now(timezone.utc).isoformat(),
        "razorpay_note": "[SIMULATED] Production uses RazorpayX Payout API",
        "guidewire_mapping": {
            "system": "BillingCenter",
            "endpoint": "POST /bc/rest/v1/payments",
            "status": "simulated"
        }
    }
    
    # Update claim status to paid
    supabase = get_supabase()
    supabase.table("claims").update({
        "status": "paid",
        "paid_at": datetime.now(timezone.utc).isoformat(),
        "transaction_id": transaction_id,
    }).eq("id", claim_id).execute()
    
    return payout_result
```

Wire into claims router: add `POST /{claim_id}/payout` endpoint that calls `process_payout()`.

<action>
Create `frontend/components/claims-timeline.tsx`:

A vertical timeline component showing claim lifecycle as WhatsApp-style notification cards. Each claim shows these stages as a vertical step progression:

1. Trigger Detected (lightning bolt icon, timestamp, trigger type + data snapshot)
2. Claim Created (file icon, claim number, payout amount)
3. Fraud Checks (shield icon, result: passed/flagged, fraud score)
4. Approved/Rejected (check/x icon, timestamp)
5. Payout Initiated (wallet icon, transaction_id)
6. Payout Completed (checkmark-circle icon, amount, "Credited to UPI")

Props interface:
```typescript
interface Claim {
  claim_number: string
  trigger_type: string
  trigger_timestamp: string
  payout_amount: number
  status: 'pending' | 'approved' | 'rejected' | 'paid'
  fraud_score: number
  approved_at?: string
  paid_at?: string
  transaction_id?: string
  created_at: string
}

interface ClaimsTimelineProps {
  claims: Claim[]
}
```

Styling:
- Dark card bg-slate-800/50, rounded-xl
- Left border color per trigger type: rain=blue-500, heat=orange-500, aqi=purple-500, bandh=red-500, compound=amber-500
- Status badges with appropriate colors (approved=emerald, pending=amber, rejected=rose, paid=emerald with check)
- Payout amount in large bold text (Rs.XXX)
- Framer-motion stagger animation on mount
- Empty state: "No claims yet. Claims are created automatically when trigger conditions are detected."
</action>
