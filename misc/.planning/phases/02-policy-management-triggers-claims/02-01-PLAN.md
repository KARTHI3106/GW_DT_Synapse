# Plan 02-01: Policy Dashboard + Management API

## Objective

Wire the existing policy dashboard UI to real backend data and add policy lifecycle management (renew, pause, resume). The dashboard currently uses hardcoded data; this plan connects it to the live Supabase backend and adds Guidewire integration badges.

## Context

See: requirements.md (FR-2.1, FR-2.2, FR-2.3)
See: design.md (Policy Dashboard wireframe, Guidewire mapping)
See: backend/routers/policies.py (existing router)
See: frontend/components/policy-card.tsx (existing component)
See: frontend/lib/api.ts (typed API client)

## Tasks

### Task 1: Consolidate policy backend service

<read_first>
- backend/routers/policies.py
- backend/services/policy_service.py
- backend/routers/workers.py (for policy creation flow reference)
</read_first>

<action>
Refactor `backend/services/policy_service.py` to be the single source of truth for policy operations:

1. Keep existing `create_policy(worker_id, premium, earnings)` function
2. Add `get_policy_detail(worker_id: str) -> dict` that returns:
   - Policy fields: policy_number, start_date, end_date, weekly_premium, coverage_amount, status
   - Computed: active_claims_count (count claims where status in approved/pending/paid), total_payout (sum payout_amount where status=paid)
   - renewal_date: end_date
3. Add `renew_policy(policy_id: str) -> dict`:
   - Recalculate premium using `ml.premium_calculator.calculate_premium`
   - Update end_date to current end_date + 7 days
   - Update weekly_premium if changed
   - Insert row into premium_history
   - Return updated policy
4. Add `pause_policy(policy_id: str) -> dict`:
   - Set status = "paused", updated_at = now
5. Add `resume_policy(policy_id: str) -> dict`:
   - Set status = "active", updated_at = now

Remove duplicate policy creation code from `backend/routers/policies.py` (if it has inline creation logic that duplicates the service).
</action>

<acceptance_criteria>
- backend/services/policy_service.py contains `def get_policy_detail(`
- backend/services/policy_service.py contains `def renew_policy(`
- backend/services/policy_service.py contains `def pause_policy(`
- backend/services/policy_service.py contains `def resume_policy(`
- No inline policy creation logic remains in policies.py router (creation delegates to service)
</acceptance_criteria>

### Task 2: Update policies router with management endpoints

<read_first>
- backend/routers/policies.py
- backend/services/policy_service.py (after Task 1)
- requirements.md (FR-2.2, FR-2.3 endpoint specs)
</read_first>

<action>
Update `backend/routers/policies.py`:

1. `GET /{worker_id}` — call `policy_service.get_policy_detail(worker_id)` to return full policy with claims summary and Guidewire metadata:
   ```python
   return {
       **policy_detail,
       "guidewire_mapping": {
           "system": "PolicyCenter",
           "endpoint": "POST /pc/rest/v1/policies",
           "status": "simulated"
       }
   }
   ```

2. `POST /{policy_id}/renew` — call `policy_service.renew_policy(policy_id)`, return new policy with updated premium

3. `POST /{policy_id}/pause` — call `policy_service.pause_policy(policy_id)`

4. `POST /{policy_id}/resume` — call `policy_service.resume_policy(policy_id)`

Add Pydantic response models:
- `PolicyDetailResponse` with all fields from get_policy_detail
- `PolicyActionResponse` with policy_id, status, message
</action>
